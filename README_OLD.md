随机国家邮件 RP 系统

内部开发 README（最终版）

一、项目定义

本项目是一个由 Faker.js 驱动 的随机世界生成与交互系统。

玩家通过 Fake Email 界面 进行唯一的读写交互，其余所有界面均为 只读。

系统支持 OpenAI API 兼容配置，具备 Setup 初始化页面 与 管理后台。

FallenJS 仅作为风格参考（边缘、破碎、地下感），不参与逻辑构建。

二、核心交互规则（请严格遵守）

1. 玩家交互边界

- 唯一可写界面：Fake Email
   - 玩家可以阅读邮件
   - 玩家可以撰写并发送回复
- 所有其他界面：只读
   - 内部群组讨论
   - 设备状态面板
   - 国家概况 / 地图
   - 天气 / 随机事件
   - 角色档案

玩家只能“旁观”和“发邮件”，不能直接编辑世界参数、角色状态或设备配置。

2. 动态角色生成（核心逻辑）

- 角色不是预设的，不使用固定 NPC。
- 采用 Faker.js 实时生成。
- 触发机制：
玩家在邮件中提及“财政部”→ 系统立即生成“财政部长”角色→ 赋予技术认知、性格、设备属性→ 存入数据库→ 后续邮件中永久复用

三、初始设备与环境配置（硬编码要求）

所有初始设备必须处于 “濒临崩溃” 状态。

1. 硬件清单

- ☀️ 快烂掉的太阳能板
- 🔋 快坏掉的户外移动电源
- 💻 Windows 3.1 电脑
- 🧱 BASIC 电脑
- 📠 垃圾调制解调器
- ☎️ 垃圾电话
- 🔌 垃圾插排

2. 故障规则

- 设备应具备极高的故障率
- 设计标准：
“一天没坏 12 次，说明还不够烂。”
- 网络延迟、断连、丢包应为常态。

四、系统架构

Setup 页面（初始配置）
        ↓
Faker.js 生成引擎（国家 / 角色 / 事件）
        ↓
Fake Email 交互（玩家唯一输入口）
        ↓
内部群组讨论生成（只读展示）
        ↓
OpenSearch 类脑知识库（持久化）
        ↓
管理后台（运维与监控）

五、功能模块说明

5.1 Setup 页面

- OpenAI API Key 配置
- API Endpoint 设置（兼容 OpenAI 格式）
- 模型选择
- 风格权重调节（FallenJS 参考）
- 随机种子（可选）

5.2 Fake Email 界面（主交互）

- 异步邮件通信
- 延迟与丢包模拟
- 流式输出（打字机效果）
- 支持多模态内容：
   - 日志片段
   - 系统报错
   - 乱码文本
   - 二进制数据

5.3 内部讨论界面（只读）

- 多角色群组聊天视图
- 展示决策过程
- 包含意见冲突与妥协
- 最终由主导角色汇总对外邮件

5.4 OpenSearch 类脑知识库

存储对象：

- 国家档案
- 动态生成的角色（含历史状态）
- 设备状态记录
- 完整邮件往来
- 国内随机事件

用途：

- 跨会话长期记忆
- 角色与剧情连贯性
- 数据回溯与查询

5.5 管理后台（Admin）

- 查看所有对话记录
- 查看角色 / 设备 / 国家状态
- 手动触发：
   - 随机事件
   - 设备状态重置
   - 系统参数调整
- 数据导出（可选）

六、开发注意事项

- ❌ 禁止预设“总统”“财政部长”等固定角色表
- ❌ 禁止构建“正常现代化国家”
- ✅ 世界应呈现：
   - 拼凑感
   - 不稳定性
   - 技术代差
   - 荒诞但自洽的逻辑
- ✅ 如果代码实现看起来“过于优雅和稳定”，请立刻引入更多随机故障

七、MVP 第一阶段范围

1. Setup 页面（API 配置）
2. Fake Email 界面（读写功能）
3. 初始“快坏设备”环境加载
4. Faker.js 动态角色生成逻辑
5. 内部群组讨论界面（只读）
6. 第一封入站邮件生成
7. OpenSearch 数据存储链路
8. 最简管理后台（查看功能）
